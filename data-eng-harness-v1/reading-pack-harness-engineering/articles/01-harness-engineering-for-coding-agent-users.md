## Resumen operativo

- Define el arnés como el conjunto de elementos que rodean al modelo en un agente de coding y distingue entre el agente base y el outer harness específico del caso de uso.
- Propone un modelo basado en guides (feedforward) y sensors (feedback) para aumentar la probabilidad de buenos resultados y habilitar autocorrección.
- Distingue controles computational e inferential y categorías como maintainability harness, architecture fitness harness y behaviour harness.
